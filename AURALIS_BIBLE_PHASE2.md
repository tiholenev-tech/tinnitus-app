убивай, свиря". Това е същинско PWA ограничение → Capacitor е истинското решение.

## 1. ДИСТРИБУЦИЯ: web-базиран продукт + Capacitor обвивка
- Сайтът (`tinnitus-app.help`) е сърцето: landing + статии + тест + плащане. SEO/GEO трафик.
- Capacitor = **server-served APK** (тънка обвивка зареждаща живия сайт). Ъпдейтваш сайта → app нов без нов APK.
- НЕ е в конфликт с web стратегията — опакова същия сайт + добавя native слой.
- Аудиторията 50+/70+ търси в Google („шум в ушите", „acufeni"), влиза през статия — не през app store. Избягваме войната с 200 еднакви апа.

### Capacitor решава ТРИ неща:
1. Заключен екран (бъг №1) — foreground service държи аудиото
2. Audio focus при обаждане — паузира при звънене, връща след разговор
3. Дистрибуция (Google Play или директен APK)

## 2. ПАЗАРИ (ред) + ЦЕНА

### Ред: България ПЪРВО → Италия ПОСЛЕ
- **България = тестов пазар:** роден език, бързо/евтино тестване, гладене на бъгове, валидиране с реални хора. НЕ за печалба (ниски доходи).
- **Италия = приходен пазар** (след изпипване на BG + качествен превод): най-застаряваща ЕС (24.7% над 65), частни здравни разходи (€560+/год), слаба конкуренция на „acufeni", нито едно водещо tinnitus app на италиански.
- Следващи кандидати: Нидерландия, Франция, Полша.
- **ИЗБЯГВАЙ:** Германия + английски (свръхконкуренция, държавни DiGA, милионни бюджети).
- Принцип: **платежоспособност > заболеваемост**. BG/RO/GR = най-тежък тинитус, но не плащат. IT/NL/FR = пари.

### Цена и модел (финално)
- **€19.99 еднократно** (НЕ абонамент). Плащаш веднъж → твое завинаги.
- **14-дневен пълен trial** в приложението → после paywall.
- **Променливи отстъпки** според конверсията (admin PHP, без код).
- Оптимизирай по **приход на посетител** (цена × плащащи), НЕ по сваляния. Не на малък трафик (нужни стотици посетители за сигнал).
- Малки FB реклами с отстъпки = ДОПЪЛНИТЕЛЕН канал към SEO.

### Безплатно vs платено
- **Landing = напълно безплатен и отворен:**
  - Статии (задължително отворени — те са SEO трафикът; зад paywall = невидими за Google)
  - Тест + аха-момент (residual inhibition) ПРЯКО на landing-а
  - **1-2 звука се чуват на самата страница** ПРЕДИ сваляне (убеждава „работи")
- **App:** тест + облекчение задължително вътре; пълен 14-дн trial → €19.99 отключва библиотека (~256 звука), миксер, дневник ЗАВИНАГИ.

## 3. CAPACITOR — статус и план

### Готово (CC, 2026-06-03):
- **Фаза 1** (PR #3, branch `feat/capacitor-android`): тънка обвивка, `mobile/` папка.
  - server-served, `server.url = https://tinnitus-app.help`, appId `help.tinnitusapp.app`, appName AURALIS.
  - GitHub Actions build (`.github/workflows/android-build.yml`) → debug APK артефакт. Java НЕ нужна локално.
  - `mobile/android/` native проект комитнат (за Фаза 2 редакции на manifest/service).

### §5 КАПАН (изяде дни на RunMyStore — ПОМНИ ГО):
- server.url + native plugins → `window.Capacitor` undefined на някои устройства (Samsung WebView auto-injection се проваля).
- Решение: хоствай Capacitor bridge файловете на сървъра + ръчно инжектиране ПРЕДИ ползване на native функция.

### Капани (общи):
- safe-area/notch → `viewport-fit=cover` + CSS `env(safe-area-inset-*)`
- кеш → force-quit + clear cache на телефона след промяна
- HTTPS задължителен (`cleartext: false`)
- Android 14: foreground service иска `foregroundServiceType` + `POST_NOTIFICATIONS` runtime permission

### Signing:
- Debug APK = тест (директна инсталация). Signed release = Play (keystore извън git).

## 6. SEO + GEO ТЕХНИЧЕСКИ ПРАВИЛА (за landing-а ОТ ДЕН 1)

### Технически (твърд факт)
- Core Web Vitals: LCP ≤2.5s, **INP ≤200ms**, CLS ≤0.1. Pitch тестът (тежък Web Audio JS) → раздробявай с `scheduler.yield()`, дебаунсвай (иначе блокира main thread, проваля INP).
- **Server-side HTML задължителен** (PHP правилен; CSR = дефект). Грешки → реален 404/500, НЕ 200 с JS грешка.
- Semantic HTML5: `main` (най-тежко), `article`, `section`, `nav`, `aside`. H1-H6 строга йерархия.

### Schema.org JSON-LD
- `Article`/`BlogPosting` (author+publisher+datePublished+dateModified, `sameAs` → УНГ рецензента)
- `FAQPage` (zero-click + гласово търсене)
- `MedicalCondition` за тинитуса. `Physician`/`MedicalOrganization` само защото ИМА реален рецензент.

### YMYL / E-E-A-T (критично — 67%+ неоптимизирани здравни сайтове загубиха видимост)
- Реален автор + „Рецензирано от" таг с УНГ специалист
- Всяко здравно твърдение → изходящ линк към PubMed/научен източник
- „За нас", реален адрес, контакти, SSL, disclaimer

### GEO / AI visibility
- **llms.txt е МЪРТЪВ** (0.1% трафик, Google не го чете) — НЕ го прави.
- AI Overviews при 15-25% заявки; нецитирани губят 18-25% трафик → GEO задължителен.
- Структура за AI цитиране: **BLUF** (първите 150-200 думи = пълен отговор), въпросителни заглавия („Как тинитусът влияе на съня?"), Q&A чънкове (първо изречение 40-60 думи директен отговор), много конкретни числа.
