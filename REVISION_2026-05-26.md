### 🔴 `e5e3abc` — FIX FLIGHT-TOKEN

**Логика:** ✅ Правилна за Layer 1.
- `++currentFlightToken` на всеки `playLayer1` / `playUrl`
- Re-check на 2 места (pre-fetch, post-decode, post-generated)
- Stale promise → return без startLayer1Source

**🔴 Bug:** `playLayer2` НЕ е защитен от същия pattern. Race scenario:
1. SEQ-REVEAL schedule-ва L2-A в `pendingL2RevealTimer`
2. Преди timer да fire-не, user отваря NoisePicker → избира L2-B
3. `onNoiseChanged` → `playLayer2('B')` директно
4. SEQ-REVEAL timer fires → `playLayer2('A')` стартира паралелно
5. Двете в fetch race. Whichever resolves LAST извиква `startLayer2Source` last → `hardStopLayer2` kills предишния. WRONG noise plays.

**Severity:** Medium — изисква user да смени noise по време на L2 delay window (2.5–3.5s след tap Play).

**Proposed fix:**
```js
var currentL2FlightToken = 0;
function playLayer2(noiseId, opts) {
  var myToken = ++currentL2FlightToken;
  ...
  return bufferPromise.then(function (buffer) {
    if (myToken !== currentL2FlightToken) {
      console.log('[playLayer2] STALE — abort');
      return;
    }
    startLayer2Source(noiseId, buffer, spec.filter || null, opts);
  });
}
```

**Comment ↔ behavior:** Coment в [audio-engine.js:201](js/audio-engine.js:201) казва "Noise pack normalized to -26 LUFS (audio_normalize.py done) — cap removed." Не verified дали normalization е реално приложена на disk. `library_staging_normalized/` съществува с 256 файла, но LUFS не е проверен. Препоръка: spot-check 5 noise файла с ffmpeg loudnorm.

### ✅ + ⚠ `40e64fc` — FIX SEQ-REVEAL-TIMERS

**Логика:** ✅ Архитектурно правилна.
- `pendingL2RevealTimer` single global tracker + cancel при нов SEQ-REVEAL — добре.
- `activeRevealRequest` counter + stale check на 3 нива (post-L1-resolve, post-L2-timer-fire, post-L2-play) — defense-in-depth.
- `lastL1Start { presetId, ts }` re-entrancy guard 500ms — belt-and-suspenders.
- L2 schedule chained към `l1Promise.then` (не от call-site) — правилно решение за L1-then-L2 ordering.
- `stopLayer1` / `pause` / `stop` всички `activeRevealRequest++` + clear pending — пълно покритие.

**⚠ Concern #1:** Re-entrancy guard leak (вече описан в Summary):
- `startLayer1Source` line 549-554 early-return при same presetId в <500ms
- `l1Promise` ВСЕ ПАК resolves (защото playLayer1Spec resolves Promise from `resumeContext().then(...)` chain)
- SEQ-REVEAL `.then` callback извиква dispatch `audio:reveal-l1` event + schedules L2 timer
- L2 starts, но L1 не свири
- User вижда: само L2 noise, без main sound

**Severity:** Low — изисква 2 rapid taps на ЕДНИЯ И СЪЩИ sound в <500ms. Рядко в реален UX.

**Proposed fix:** `startLayer1Source` да return-ва индикатор (e.g. `false`) и `playLayer1Spec` да го пропагира към promise rejection, или `l1Promise` да resolves САМО след successful start. Алтернатива — добавяне на check в SEQ-REVEAL `.then`:
```js
l1Promise.then(function () {
  if (!layer1.source || layer1.presetId !== soundId) {
    console.log('[seq-reveal] L1 not actually playing — skip L2');
    return;
  }
  ...
});
```

**⚠ Concern #2:** `playSequentialReveal` line 423-425 — `hardStopLayer2()` unconditionally stops L2 преди да започне нов reveal, дори ако noiseId е СЪЩИЯТ (например user отваря player на различен sound, но SS_R профил → same brown_lp500). User чува momentary L2 silence, после fade-in пак. Малък UX hiccup. Severity: Cosmetic.

### 🔴 #3 — Path mismatch documentation

| Where | Path |
|---|---|
| `audio-engine.js` (HEAD code) | `library_staging_normalized/` |

### ⚠ — Master volume conflict at sleep timer

`applyProfileConfig` (player.js:569-571) извиква `setMasterVolume(cfg.masterVol)` при Player.open. Ако sleep timer е в fade-out фаза (`sleepStopTimerId` active), masterGain ramps to 0; setMasterVolume cancel-ва това и сетва на cfg.masterVol → sleep timer тихо се счупва. Edge case (рядко в реален UX — user не открива Player по време на sleep fade), но fragile.

**Severity:** Cosmetic edge case. No fix needed now, бележка за бъдеще.
