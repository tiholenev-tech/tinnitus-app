## Двете скорости
- **Качество (на ръка — Claude/човек):** BG ✅ · RO · GR — рекламните пазари (Пласт 2).
- **Машина (DeepL — Тихол пуска → Claude монтира):** всички останали Stripe езици (Пласт 1, пасивен доход).

## ГЕЙТОВЕ (важат за ВСЕКИ език, без изключение)
1. **`python3 tools/i18n_validate.py <lang>` = 0 проблема** — структурен mount-гейт: пълна 1:1 структура с `bg.json`, без липсващи ключове, без изпуснати плейсхолдъри `{n}` / backtick-ID-та / emoji, без Cyrillic-leak. (Хваща типичните грешки на DeepL.)
2. **`python3 tools/check_claims.py <lang>` = 0 BLOCK** — медико-правна защита (никъде „лекува/cura/diagnostica" като твърдение на продукта). ℹ️ review = образователно, ОК.
3. **Version bump** при пипане на app-shell JS (виж А4).
4. **Не чупи бащата** — paywall стои дормантен, заварените grandfather-нати.

## A. ПРИЛОЖЕНИЕ (вече drop-in готово — i18n архитектурата съществува)
Стъпки за нов app-език `xx`:

1. **DeepL: `i18n/bg.json` → `i18n/xx.json`.** Внимание при DeepL:
   - ползвай **JSON-aware** режим (не плосък текст) — да не счупи структурата;
   - **НЕ** превеждай плейсхолдърите: `{n} {total} {freq} {delta} {goal}`;
   - **НЕ** превеждай ID-та в backticks: `` `pink_lowpass_4000` ``, `` `brown_lowpass_500` ``, `` `daily` ``, `` `relaxation` ``, `` `sleep_deep` `` и т.н.;
   - запази emoji-заглавията (📊 🎯 💡 📈 🧠 📚) и реалните заглавия на публикации (Pantev, Newman…);
   - `_meta`: сложи `locale`, `name`, `rtl` (true само за ar/he).
2. `js/i18n.js` → добави `'xx'` в `var SUPPORTED`.
3. `js/language-picker.js` → добави `{ code:'xx', name, native, flag }` в `LANGUAGES`.
4. **VERSION BUMP** (стъпки 2–3 пипат кеширани shell файлове): бъмпни `VERSION` (`service-worker.js`) **И** `CODE_VERSION` (`app.html`). `i18n/xx.json` сам по себе си е staleWhileRevalidate (без bump) — bump-ът е заради `SUPPORTED`/`LANGUAGES`.
5. **Двата гейта:** `python3 tools/i18n_validate.py xx` (0 проблема) **И** `python3 tools/check_claims.py xx` (0 BLOCK).

> Разделение: **Тихол** дава `xx.json` от DeepL; **Claude** прави 2–5 + bump + двата гейта. (RO/GR ги превежда Claude на ръка за качество.)

## B. САЙТ (скелет — следваща задача, преди дългата опашка)
**Текущо (не скалира):** BG = `inc/site.php`, IT = `inc/site-it.php` + `it/` — по един PHP слой на език, hardcoded `hreflang` само BG↔IT.

**Цел (data-driven скелет):**
- централен `$LANGUAGES` регистър (code, native, флаг, url-префикс, dir);
- helper, който авто-генерира **реципрочния hreflang клъстер** (всяка страница изброява всички езици) — това е search-приоритетът;
- секции/статии/UI като **данни на език** (JSON), един общ renderer.

Тогава **нов сайтов език = пусни преведения data-файл + 1 ред в `$LANGUAGES`**; hreflang и sitemap се авто-обновяват.

**Обхват за дългата опашка (Пласт 1):** минимумът е **начална + ключови Q&A статии** (конверсия + GEO странична врата). Пълните 20 статии са само за рекламните (Пласт 2: BG/RO/GR).
