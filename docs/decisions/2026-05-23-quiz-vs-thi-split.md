# DECISION: Quiz + THI разделени във времето

**Дата:** 23.05.2026
**Решено от:** Тихол (founder) + Claude (Opus 4.7)
**Статус:** ✅ Заключено

## Контекст

В научните документи има два различни въпросника:

1. **Quiz (15 въпроса, 4-5 опции)** — от `01-quiz-15-questions-validated.md`
   - Цел: персонализация (избор на звуков микс според 5 фенотипа)
   - Изход: профил (TH_C / DN_S / SS_R / SM_F / HB_M) + Distress Index 0-20
   - Време: 5-7 минути

2. **THI (25 въпроса, 3 опции Да/Понякога/Не)** — от `18-success-measurement-thi-tfi.md`
   - Цел: клиничен baseline + измерване на прогрес
   - Изход: 0-100 точки → 5 степени тежест
   - MCID = 7 точки (минимална клинично значима промяна)
   - Време: 5 минути

## Обсъдените опции

- **Опция A:** Само Quiz (15 въпр.)
- **Опция B:** Само THI (25 въпр.)
- **Опция Z:** И двете, разделени във времето ✅ **избрана**

## Решение

**Опция Z — двата теста, разделени:**

| Ден | Действие | Защо |
|---|---|---|
| Ден 0 | Welcome + Quiz (15 въпр.) + FIRST WIN (звуков микс пуска 2 мин) | Персонализация + изграждане на доверие. THI е "тежко" (катастрофизиращи въпроси) — отлага се. |
| Ден 2-3 | THI baseline (25 въпр.) | Доверието вече е изградено. Готов да отговори на тежки въпроси. |
| Ден 14 | THI повторно | Сравнение → проверка дали е постигнат MCID (-7 точки) |
| Ден 30 | THI повторно | Дългосрочна валидация |

## Обосновка (от научните документи)

- **`03-onboarding-strategy-50plus.md`:** "Първият FIRST WIN момент трябва да се случи преди да поискаме нещо тежко"
- **`15-churn-analysis.md`:** Ден 1 retention за wellness = 25-30%. 40 въпроса на първото отваряне = гарантирано отпадане при 50+
- **`14-sensory-behavioral-dynamics.md`:** Първи 30 сек = решават дали остава потребителят
- **`08-cbt-validated-digital-protocols.md`:** Kalmeda прави диагностика **след** изграждане на терапевтичен съюз — не преди

## Защо НЕ само Quiz?

- AURALIS_HANDOFF_v5.md Част 13 изисква **THI: 62 → ≤55 (-7 точки)** като метрика #1 за успех с бащата
- Subjective 0-10 slider няма клинична тежест за N=1 case study (PR кампания)
- THI е златен стандарт (α на Кронбах >0.90)

## Защо НЕ само THI?

- Без персонализация (всички получават един и същ микс) — лош UX за wellness
- 5-те фенотипа изискват специфичен Quiz (не THI)
- AI assistant архитектурата (`05-ai-recommendations-architecture.md`) се базира на фенотипи от Quiz

## Импликации за разработка

1. **Quiz wizard** — приоритет 1 за Седмица 1
2. **THI компонент** — отделен component, активира се на Ден 2-3 via push notification
3. **DB schema** — нужни 3 таблици:
   - `quiz_results` (профил, DI, дата)
   - `thi_results` (timestamp, общ score, 25 въпроса JSON)
   - `thi_history` (за proget tracking)
4. **Прогрес визуализация** — chart с THI scores по дни 0/14/30
