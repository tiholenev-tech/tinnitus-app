## 🔴 P0 — CRITICAL

### Audio Engine

**P0-1 · audio-engine.js:~142** — AudioContext resume swallow
- `ctx.resume()` без `await`, `.catch()` swallow-ва suspension rejection
- **Симптом:** iOS AudioContext остава suspended, play() queue-ват се завинаги
- **Verify:** `grep -n "ctx.resume" js/audio-engine.js`

**P0-2 · audio-engine.js:~1163** — disconnect преди stop
- `src.disconnect()` извикан ПРЕДИ `src.stop()` в stopLayer1()
- **Симптом:** Safari/iOS не позволява stop() след disconnect → абнормално прекъсване
- **Verify:** Намери stopLayer1() функцията, провери order на operations

**P0-3 · audio-engine.js:~608** — pendingL2RevealTimer leak
- Не се cancel-ва в stopLayer1() при switch на sound
- **Симптом:** L2 fires на стария sound 30-60s след switch (audio leak)
- **Verify:** Търси `pendingL2RevealTimer` и виж дали се clearTimeout-ва във всички paths

**P0-4 · player.js:~743** — Promise rejection без user notification
- `AudioEngine.play()` rejection swallow-нат
- **Симптом:** UI показва pause, audio мълчи, user re-tap-ва → race
- **Verify:** Намери `AudioEngine.play(` calls в player.js, виж .catch handlers

### Data Loss / Persistence

**P0-5 · index.html:271** ✅ VERIFIED от мен директно
- `s.resetQuiz()` извикан при brand-home click ако onboarding incomplete
- **Симптом:** Изтрива partial quiz progress дори при случаен click на logo-то
- **Виж директно:** index.html lines 269-272

**P0-6 · quiz.js:~373 / onboarding.js:~374** — consent не persist
- `consentGranted` set в memory, не serialize-ван в localStorage
- **Симптом:** Reload по средата → user трябва да re-check consent
- **Verify:** Търси `consentGranted` setters, виж дали извикват state.save()

**P0-7 · settings.js:~855** — deleteAllData incomplete
- Match-ва само `auralis_` / `auralis-` prefix
- **Симптом:** Orphan keys (theme, locale) остават → privacy claim violation
- **Verify:** Прочети deleteAllData() и сравни с реално запазени keys

### Service Worker / Deploy

**P0-8 · service-worker.js:~22-25** — **CACHE_AUDIO не bump-нат** 🔥
- CACHE_AUDIO остава `v3` докато SHELL версия е `1.0.52`
- **Симптом:** Audio engine fixes НЕ достигат phone test без manual cache clear
- **Verify:** `grep -n "CACHE" service-worker.js` — провери дали v3 е до сега
- **Препоръка:** Това обяснява защо последните audio fixes не се виждат

**P0-9 · index.html:293** ✅ VERIFIED — silent SW catch
- `.catch(function(){})` — нямa лог при SW грешка
- **Виж директно:** index.html line 293

### State / Race

**P0-10 · favorites.js:~219-248** — listener leak в showSheet render
- Sort/action buttons re-bind на всеки re-render без cleanup
- **Симптом:** Rapid clicks → duplicate play/remove actions
- **Verify:** Намери showSheet() и виж дали render() пресъздава listeners

**P0-11 · onboarding-tour.js:~79-86** — null querySelector crash
- `document.querySelector(step.selector)` връща null → `getBoundingClientRect()` crash
- **Симптом:** Tour crash mid-step ако target element още не е в DOM
- **Verify:** Search `querySelector` в onboarding-tour.js, виж null checks

**P0-12 · category-info-sheet.js:~396** — scroll lock не release
- `.cis-locked` class на body не remove-нат при close
- **Симптом:** Body scroll остава lock-нат за всички следващи modals
- **Verify:** Виж close()/buildFallbackOverlay() — има ли classList.remove

### PWA Install

**P0-13 · manifest.json + icons/** ✅ VERIFIED от git status
- `icons/icon-*.png` са D (deleted), реалните са в `app-icons/`
- **Симптом:** Fresh clone → 404 → PWA install fail
- **Виж директно:** git status, и manifest.json icons[] paths

**P0-14 · i18n.js:~65-69** — RTL само dir attribute
- Без CSS layout flip → Arabic UI broken (text-align, числа)
- **Verify:** Виж `setLocale` за RTL handling, провери CSS за `[dir="rtl"]` selectors
