## ⚙️ НЕЗАМЕНИМИ технически правила (наруши ги → чупиш продукта)
1. **Развивай на branch `claude/<твоя>` ; НЕ push-вай в `main` директно.** PR →
   draft → merge. (Сървърът авто-дърпа `main` → деплой ~1 мин. Нямам droplet достъп.)
2. **Version bump:** при промяна на КЕШИРАН app-shell файл (JS/HTML/CSS в
   `service-worker.js` → `SHELL_FILES`) бъмпни **И двете** — `VERSION`
   (`service-worker.js`) **И** `CODE_VERSION` (`app.html`). Иначе старият кеш
   се сервира и промените са невидими на устройството.
3. **Без version bump за:** `i18n/*.json` (staleWhileRevalidate) и публичните
   PHP/XML страници (`/`, `/articles/`, `/it/`, `sitemap.xml` — networkFirst).
4. **Бащата на Тихол има инсталирано приложение — не го чупи.** Paywall стои
   дормантен; заварените са grandfather-нати; standalone-redirect пази инсталациите.
5. **i18n:** `it.json` трябва да е ПЪЛЕН копие-структура на `bg.json` (иначе
   `t()` пада на ключа). Превеждай секция по секция, валидирай leaf-паритет,
   проверявай за Cyrillic-leak.
6. **Repo scope:** само `tiholenev-tech/tinnitus-app`. Модел-идентификатори НЕ
   влизат в commit/PR/код.
